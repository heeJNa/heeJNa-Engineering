# [도메인] 검증 체크리스트

## 트리거
- 파일 패턴: `*/<pattern>`
- 훅 메시지에 핵심 체크리스트가 직접 주입됨

## 상세 체크리스트

- [ ] 항목 1 — 설명
- [ ] 항목 2 — 설명
- [ ] 항목 3 — 설명

## 위반 시 수정 방법

| 위반 | 수정 |
|------|------|
| 위반 예시 1 | 수정 방법 1 |
| 위반 예시 2 | 수정 방법 2 |

## 참조
- 규칙 출처: `path/to/rules.md`

---

## 커스터마이징 가이드

### 새 검증 도메인 추가 방법

1. 이 템플릿을 복사하여 `skills/verify/<domain>.md` 생성
2. `hooks/domain-verify.sh`에 case 블록 추가:
   ```bash
   # ── 새 도메인 ──
   */path/to/pattern*.ext)
     MSG="[Domain Verify: 도메인명] '${FILENAME}' 검증:
   1. 핵심 규칙 1
   2. 핵심 규칙 2
   위반 발견 시 즉시 수정하세요." ;;
   ```
3. 스킬 파일에 상세 규칙, 예시, 수정 방법 기술
